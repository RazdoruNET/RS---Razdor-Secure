# DPI-Bypass Launcher

🔥 **Полная система обхода DPI для маршрутизации всего трафика через мультицепочечный обход**

## 🚀 Быстрый запуск

```bash
# Запуск с автоматической настройкой
sudo ./start_dpi_bypass.sh

# Проверка статуса
./status_dpi_bypass.sh

# Остановка
./stop_dpi_bypass.sh
```

## 📋 Компоненты системы

### 1. **launch_dpi_bypass_proxy.py** - Основной прокси-сервер
- Маршрутизирует весь трафик через DPI-Bypass Combiner v2.0
- Поддерживает HTTP/HTTPS прокси на порту 8080
- Автоматически определяет необходимость обхода
- Встроенная статистика и веб-интерфейс

### 2. **setup_system_proxy.py** - Настройка системного прокси
- Автоматическая настройка прокси в macOS
- Поддержка HTTP/HTTPS прокси
- Настройка доменов для обхода
- Восстановление настроек

### 3. **Скрипты управления**
- `start_dpi_bypass.sh` - Полный запуск системы
- `stop_dpi_bypass.sh` - Остановка и очистка
- `status_dpi_bypass.sh` - Проверка статуса

## 🎯 Принцип работы

### Цепочки обхода:
1. **Гос-Фрагмент** - TCP_Window_Size(1) + SNI_Spoofing(gosuslugi.ru) + TLS_Header_Junk
2. **Медиа-Хамелеон** - HTTP2_Multiplexing + SNI_Spoofing(rutube.ru) + Fake_Padding  
3. **Финансовый Призрак** - SNI_Spoofing(sbp.nspk.ru) + Encrypted_Payload + TTL_Lowering
4. **Life-Line** - Мимикрия под медицинские CGM-системы

### Whitelist Engine:
- **GOVERNMENT**: gosuslugi.ru, nalog.gov.ru, zakupki.gov.ru
- **FINANCIAL**: sbp.nspk.ru, vtb.ru, alfabank.ru, sberbank.ru
- **TELECOM**: rutube.ru, smotrim.ru, 1tv.ru, vk.video
- **TRANSPORT**: rzd.ru, yandex.ru, taxi.yandex.ru
- **MEDICAL**: librelink.com, medtrum.com, anytime.cgms.ru

## 🔧 Ручная настройка

### 1. Запуск прокси-сервера
```bash
python3 launch_dpi_bypass_proxy.py --port 8080 --log dpi_bypass.log
```

### 2. Настройка системного прокси
```bash
# Настройка
sudo python3 setup_system_proxy.py --port 8080

# Проверка статуса
python3 setup_system_proxy.py --status

# Отключение
sudo python3 setup_system_proxy.py --disable
```

### 3. Настройка браузера
- **HTTP Proxy**: 127.0.0.1:8080
- **HTTPS Proxy**: 127.0.0.1:8080
- **Исключения**: localhost, 127.0.0.1, *.local, *.ru

## 📊 Статистика и мониторинг

### Веб-интерфейс:
- **URL**: http://127.0.0.1:8080/stats
- **Информация**: количество запросов, успешные обходы, время работы
- **Статус**: активные цепочки, текущие маски

### Лог-файлы:
- **Расположение**: `logs/`
- **Формат**: `dpi_bypass_YYYYMMDD_HHMMSS.log`
- **Содержимое**: детальная информация о запросах и обходах

## 🛡️ Безопасность и приватность

### Особенности:
- **Zero-Dependency**: не требует внешних зависимостей
- **Local Processing**: весь трафик обрабатывается локально
- **Whitelist Mimicry**: мимикрия под легитимные сервисы РФ
- **Auto-Switching**: автоматическое переключение цепочек
- **Life-Line Mode**: аварийный режим через медицинские системы

### Защита от обнаружения:
- **SNI Fragmentation**: разбиение SNI на 2-3 части
- **Header Obfuscation**: обфускация HTTP заголовков
- **TLS Junk**: добавление мусорных TLS записей
- **Fake Padding**: фейковые padding данные
- **TTL Lowering**: низкий TTL для пакетов-пустышек

## ⚠️ Важные замечания

### Требования:
- **macOS**: для автоматической настройки системного прокси
- **Python 3.9+**: для работы скриптов
- **Права администратора**: для настройки системного прокси

### Ограничения:
- Работает только с HTTP/HTTPS трафиком
- Требует настройки прокси в приложениях
- Некоторые сайты могут определять использование прокси

### Рекомендации:
- Используйте совместно с HTTPS Everywhere
- Настройте исключения для локальных ресурсов
- Периодически проверяйте статистику обхода

## 🚨 Поиск проблем

### Прокси не отвечает:
```bash
# Проверка статуса
./status_dpi_bypass.sh

# Перезапуск
./stop_dpi_bypass.sh && ./start_dpi_bypass.sh
```

### Проблемы с доступом:
```bash
# Проверка логов
tail -f logs/dpi_bypass_*.log

# Тестовый запрос
curl --proxy http://127.0.0.1:8080 http://httpbin.org/ip
```

### Системный прокси:
```bash
# Сброс настроек
sudo python3 setup_system_proxy.py --disable

# Ручная настройка через Системные настройки → Сеть → Прокси
```
